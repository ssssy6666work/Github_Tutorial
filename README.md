# Git & GitHub 版本控制學習與實作

本專案記錄 Git、GitHub 與 SourceTree 的基礎學習與實作內容，從版本控制概念、Repository 建立、Commit、Push，到 Branch、Merge、Conflict、Git Flow，以及 Revert、Reset、Rebase 等進階操作。

---

## 學習內容

### Module 1｜認識版本控制

- 課程介紹
- 為何需要版本控制
- Git 下載及安裝
- Git 用途簡介
- 註冊 GitHub 帳號

學習 Git 版本控制的基本概念，了解為什麼開發專案需要保留版本紀錄，以及 Git 與 GitHub 在開發流程中的用途。

---

### Module 2｜認識 GitHub

- GitHub 介面介紹
- 創建 Repository
- Git 與 GitHub 的關係
- 製作金鑰

學習 GitHub Repository 的建立與基本操作，了解本地端 Git 與遠端 GitHub Repository 之間的關係，以及使用金鑰進行連線與驗證。

---

### Module 3｜Git 基本操作

- 常用終端機指令
- 將 Repository 下載至本地端
- 安裝 SourceTree
- SourceTree 介面介紹

練習使用終端機操作 Git，將 GitHub Repository 下載至本地端，並使用 SourceTree 圖形化工具管理 Git Repository。

---

### Module 4｜Git 版本建立

- 建立新版本
- `.gitignore` 文件
- 版本切換
- 上傳 GitHub

練習建立 Git 版本紀錄、使用 `.gitignore` 排除不需要追蹤的檔案、切換不同版本，以及將本地端程式碼 Push 至 GitHub。

---

### Module 5｜Git 分支操作

- 分支用途
- 建立及切換分支
- 合併分支 Merge
- 衝突解決
- 刪除分支

學習 Branch 的用途與基本操作，練習建立、切換、合併及刪除分支，並處理多人開發時可能發生的 Merge Conflict。

---

### Module 6｜Git 開發流程

- 分支名稱介紹
- Git Flow
- 實作：模擬多人開發

了解常見的 Branch Naming 與 Git Flow 開發流程，並透過模擬多人協作練習不同分支之間的開發與整合方式。

---

### Module 7｜Git 進階操作

- Git Revert
- Git Reset
- Git Revert / Reset / Rebase

學習 Git 版本復原與歷史調整相關操作，了解 `revert`、`reset`、`rebase` 的用途與差異。

---

## 實作內容

本 Repository 保留課程中的 Git 操作練習檔案，用於練習：

- Repository 建立與管理
- Add / Commit / Push
- `.gitignore`
- Clone 與本地端 Repository 操作
- Branch 建立與切換
- Merge 分支
- Merge Conflict 解決
- Git Flow 與多人開發流程
- Revert / Reset / Rebase
- SourceTree 圖形化 Git 操作
